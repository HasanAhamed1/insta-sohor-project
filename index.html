How does JavaScript work?

Ans: The browser compiles JavaScript code through the V8 engine. First, JS parse the source code to create a syntax tree. We know that JS is the interpreter language, the interpreter means it executes line by line. But JS also uses the Just In Time compiler to execute faster. JS then optimizes the code and converts the code to bytecode, which the computer can use.


What does JavaScript event loop do?

Ans: To execute JavaScript creates a loop. This loop is known by Event loop. JavaScript has Stack and Queue. First JS executes commands serially which are in stack. When there are no commands in stack, js takes commands from queue. This loop is known as event loop.


Write differences between Local Storage and Session Storage.

Ans: Local Storage and Session Storage both store key or values for a particular domain.

But local storage store data for long time. When browser is closed or reopened, still the data is saved. As a example, if we mute volume in youtube and close youtube, when we reopen youtube we can see that volume is still muted. Because data for mute volume was stored in local storage and it saved data.

On the other hand, session storage keeps data only for that time when the browser is opend. If we close or reopen browser session storage makes new session to store dataa, and doesn't keep old data.
